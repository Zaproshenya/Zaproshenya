# Знайдені баги після рефакторингу

## Баг 1: sendFriendRequest — зламана перевірка дублікатів

**Проблема:** Шукає `toUid` в notifications, але поле `toUid` не додається в об'єкт notification. Перевірка завжди повертає false → користувач може надіслати кілька однакових запитів.

**Крім того:** Шукає в `notifications/{fromUid}` (відправник), а notification зберігається в `notifications/{toUid}` (отримувач).

**Фікс:** Змінити запит на `notifications/{toUid}` та шукати `fromUid`:

```js
// Було (неправильно):
const notifsSnap = await db().ref('notifications/' + fromUid)
  .orderByChild('type').equalTo('friend-request').get();
notifsSnap.forEach(c => { if (c.val().toUid === toUid) alreadySent = true; });

// Стало (правильно):
const notifsSnap = await db().ref('notifications/' + toUid)
  .orderByChild('type').equalTo('friend-request').get();
notifsSnap.forEach(c => { if (c.val().fromUid === fromUid) alreadySent = true; });
```

## Баг 2: acceptFriendRequest — зайвий remove

**Проблема:** Видаляє `friend-requests/{myUid}/{fromUid}` але в новому notification-based flow ці записи ніколи не створюються.

**Фікс:** Видалити рядок `await db().ref('friend-requests/' + myUid + '/' + fromUid).remove();`

## Баг 3: acceptReq в friends.js — видаляє notification

**Статус:** Працює коректно — `deleteNotificationsByPayload` видаляє friend-request notification після прийняття.

**Ніякого впливу на правила немає.** Обидва фікси — ТІЛЬКИ код:
